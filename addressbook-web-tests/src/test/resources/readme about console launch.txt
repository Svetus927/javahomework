конфигурац файлы testng-...xml  сод инфу что запускать причем порядок тестов (тестовых групп) не определен, а порядок выполнения
 классов в них как указано так и будет
Из консоли запускать проект так gradlew test ( предвариетльно локализовавшись  в нужной папке и имея wrapper для gradle, НО
по умолчанию gradle ищет тесты с использованием фреймворка junit, поэтому обязатеьно н прописать в build.gradle строчки
test  {useTestNG() }
Если мы хотим сделать две отд задачи  например для групп и контактов и запускать их по отдельности то нужно прописать не test и т.д.
а
task testgroups(type: Test) {
  useTestng {
   suites '...xml'
  }
}
и  тогда из ком строки м выполнять отдельно задачу
gradlew testgroups

Если вып-ие из ком строки по каким то причинам не идет то вып команду
gradlew cleanTest test
